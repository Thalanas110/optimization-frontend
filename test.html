03 / 20
Optimization / Local Search

Start somewhere. Improve nearby. Stop when improvement gets hard.

  • Begin with an initial solution, often random or heuristic.
  • Evaluate nearby alternatives instead of searching the entire space.
  • Iteratively move toward a better solution.
  • Prioritize speed and practicality over guaranteed optimality.
Optimization vs Systematic Search

Systematic search

Explores many possibilities exhaustively. Higher confidence, much higher cost.

Optimization

Searches strategically for strong solutions fast. Lower certainty, better scalability.

Optimization is about finding a very good answer before time or complexity runs out.
04 / 20
Hill Climbing

Greedy local improvement

Always take the best immediate uphill move, then repeat until the search cannot improve anymore.

Core rule One solution. One neighborhood. One best next step.

Fast and intuitive, but limited by what the current path can see.

1

Start

Pick an initial state to begin the search.

2

Inspect

Evaluate the neighboring states around it.

3

Move

Choose the best improving neighbor.

4

Repeat

Keep climbing while the value improves.

5

Stop

End when no neighbor offers a better move.

Simple

Easy to explain, code, and reason about.

Fast

Works well on small or smooth search spaces.

Greedy

Chooses the best visible move, which is why it can miss the global optimum.

05 / 20
Hill Climbing Problems and Variants

Why it gets stuck

  • Local optima: a peak that is good, but not best.
  • Plateaus: flat regions with no clear uphill move.
  • Ridges: narrow improvement paths that greedy steps miss.

Variants

Steepest-ascent: evaluate all neighbors First-choice: take the first improvement Stochastic: pick among improving moves randomly Random-restart: try many different starting points

Simulated Annealing

A smarter search because it sometimes accepts worse moves on purpose.

That “bad move” is not a mistake. It is a temporary gamble that helps the algorithm escape local optima and explore somewhere better.

Key idea: controlled randomness instead of pure greed
Early in the search, it is curious. Later in the search, it becomes selective.

The method is inspired by annealing in metallurgy, where materials are heated and cooled to settle into lower-energy states.

07 / 20
Metallurgy Analogy
1

Heat

At high temperature, particles move freely and the search explores many random directions.

→
2

Mix

Temporary disorder allows worse moves, helping the algorithm escape traps and search new regions.

→
3

Cool

High T Low T

A cooling schedule gradually reduces randomness, so fewer risky moves are accepted over time.

→
4

Settle

At low temperature, the system becomes stable and the algorithm focuses near stronger candidate solutions.

08 / 20
Acceptance of Worse Solutions

Better moves are always accepted. Worse moves are accepted with a probability.

  • f(s') measures the new state.
  • f(s) measures the current state.
  • T controls how willing the algorithm is to take risk.

What this changes

  • Hill Climbing says “never go down.”
  • Simulated Annealing says “go down sometimes if it may help you reach a better hill.”
  • The lower the temperature, the harder it is for a worse move to be accepted.
09 / 20
Exploration vs Exploitation
High Temperature

Exploration

  • Worse moves are accepted more often.
  • The search spreads across more regions.
  • Useful for escaping local optima early.
Low Temperature

Exploitation

  • Worse moves become unlikely.
  • The algorithm behaves more like Hill Climbing.
  • Useful for refining strong candidates late in the run.

Genetic Algorithms

Instead of improving one solution, evolve many candidate solutions together.

Genetic Algorithms borrow the logic of natural evolution: keep better candidates, combine them, mutate them, and repeat across generations.

Population

A set of possible solutions.

Fitness

A score telling how good each solution is.

Selection

Prefer stronger candidates for reproduction.

Generations

Repeat the cycle until a good stopping point.

11 / 20
Selection, Crossover, Mutation, Generations

Selection

Solutions with higher fitness are more likely to pass their traits forward.

Crossover

New solutions inherit useful characteristics from parent solutions.

Mutation

Random changes keep diversity alive and prevent premature convergence.

Evaluate → Select → Recombine → Mutate → Repeat

A generation is one full evolutionary cycle. More generations mean more chances to discover strong combinations.

Search Style Comparison
Hill Climbing / Simulated Annealing

Single-point search

  • Start from one solution.
  • Move through neighboring states step by step.
  • Exploration depends on the path that one solution takes.
HC: always take the better neighbor SA: sometimes accept worse moves Lower memory cost Can be trapped by local structure
13 / 20
Genetic Algorithms

Population-based search

  • Maintain many candidate solutions at once.
  • Search several regions of the space in parallel.
  • Combine strong traits instead of improving only one path.
Wider exploration More robust to local traps Higher computational cost Needs parameter tuning
14 / 20
Mental Picture

HC/SA is one explorer. GA is many explorers.

  • HC/SA: one explorer climbs a landscape, adapting as it moves.
  • GA: many explorers search different regions at the same time.
  • The more rugged the landscape, the more valuable population-based exploration becomes.
15 / 20
When to Use Hill Climbing

Use it when...

  • The search space is relatively small.
  • You need a simple and very fast method.
  • The landscape does not contain many deceptive local optima.

Advantages

  • Easy to implement
  • Low computational overhead
  • Good baseline algorithm

Main limitation

It can stop early at the wrong peak.
16 / 20
When to Use Simulated Annealing

Use it when...

  • The search space is large.
  • There are many local optima.
  • You need better exploration than Hill Climbing can offer.

Key strength

It can temporarily get worse in order to eventually get better.

Typical problems

  • Traveling Salesman Problem
  • Scheduling and timetabling
17 / 20
When to Use Genetic Algorithms

Use it when...

  • The problem is very complex.
  • The search space is huge and rugged.
  • Exploring many candidate solutions is more valuable than refining one path.

Advantages

  • Explores many regions at once
  • Less likely to get trapped by local optima
  • Works well for highly combinational problems

Trade-offs

  • Slower and more computationally expensive
  • Needs tuning for population size and mutation rate
  • Useful in engineering design, ML tuning, and drug discovery

Coding Demo

Simulated Annealing Pseudocode

A compact sketch of how temperature guides search from exploration to refinement.

Pseudo-code
function simulatedAnnealing(state, temperature):    best = state    while temperature > 0:        neighbor = randomNeighbor(state)        delta = score(neighbor) - score(state)        if delta > 0 or random() < exp(delta / temperature):            state = neighbor        if score(state) > score(best):            best = state        temperature = temperature * coolingRate    return best

Conclusion

Choose the algorithm that matches the search landscape.

Hill Climbing

Best when you need speed, simplicity, and the landscape is not too deceptive.

Simulated Annealing

Best when local traps are common and some controlled randomness can help.

Genetic Algorithms

Best when the problem is large, complex, and benefits from parallel exploration.
